# Отчёт о техническом аудите CoreAsset: База данных Supabase и Кодовая база (Выполнено ✅)

Этот отчёт содержит обновлённые результаты технического аудита базы данных Supabase и репозитория Next.js для проекта CoreAsset. Все ранее выявленные критические проблемы безопасности, функциональные ограничения и архитектурные недостатки были успешно устранены.

---

## 1. Статус аудита базы данных Supabase

### 1.1. Row Level Security (RLS) и доступ к данным — **[РЕШЕНО ✅]**
> [!NOTE]
> **Устранена уязвимость безопасности: Политики `Dev Access` отключены**
>
> Все временные политики вида `Dev Access [TableName]`, предоставлявшие неавторизованным пользователям полный доступ к таблицам, были полностью удалены.
>
> **Результат:** Вместо них настроены строгие разграничения прав на основе ролей пользователей (RLS):
> - Администраторы и IT-специалисты имеют полный CRUD-доступ к компьютерам, сотрудникам, лицензиям и инцидентам.
> - Обычные сотрудники могут просматривать только те компьютеры, которые назначены им лично, а также создавать и просматривать инциденты только от своего имени (`employee_id = auth.uid()`).

### 1.2. Триггеры и подсчет занятых лицензий (`used_seats`) — **[РЕШЕНО ✅]**
> [!NOTE]
> **Атомарный контроль на уровне СУБД**
>
> Проблема рассинхронизации счетчиков решена переносом логики на сторону PostgreSQL:
> - На таблице `computer_licenses` привязан триггер `trg_computer_licenses_seats`, который автоматически увеличивает счетчик `used_seats` в таблице `licenses` при связывании ПО с ПК и уменьшает его при удалении связи.
> - На таблицу `licenses` добавлено строгое `CHECK`-ограничение: `CHECK (used_seats <= total_seats)`. Любая попытка установить ПО сверх лимита блокируется на уровне транзакции БД.
> - Из Server Actions (`lib/actions/licenses.ts`) полностью удалено ручное инкрементирование/декрементирование, что исключает состояние гонки (Race Condition).

### 1.3. Актуальность схемы данных — **[РЕШЕНО ✅]**
* Таблицы `workplaces`, `software`, `license_pools` и `software_installations` окончательно исключены из СУБД.
* Вся кодовая база Next.js полностью переведена на новые таблицы `licenses` и `computer_licenses`, обеспечивающие оптимальную нормализованную схему M:N.

## 2. Статус аудита функций Next.js и покрытия UI

### 2.1. Шаблоны сборок ПК (`computer_templates`) — **[РЕШЕНО ✅]**
* **Интерфейс управления:** Реализован полный CRUD для шаблонов по адресу `/templates` (создание, просмотр деталей, изменение и удаление).
* **Интеграция с ПК:** Поле `template_id` выведено в UI добавления и редактирования компьютера. При выборе шаблона характеристики процессора, ОЗУ, диска, видеокарты и тип устройства предзаполняются дефолтными значениями автоматически.
* **Server Actions & Schemas:** Добавлены Zod-схемы валидации и CRUD Server Actions для шаблонов в файле `lib/actions/computer_templates.ts`.

### 2.2. Отображение лицензионных ключей (`licenses.license_key`) — **[РЕШЕНО ✅]**
* **Создание и редактирование:** Ключ лицензии запрашивается в диалоге создания.
* **Отображение и безопасность:** В списке лицензий `LicensesClientView.tsx` ключ выводится с маскировкой по умолчанию (`••••-••••-••••-••••`). Рядом расположена интерактивная кнопка показа (иконка "Глаз" / `Eye` / `EyeOff`) и быстрого копирования в буфер обмена.

### 2.3. Интеграция ролей пользователей (`employees.role`) — **[РЕШЕНО ✅]**
* Поле роли (`admin`, `employee`, `it_specialist`) полностью интегрировано:
  - В схемах валидации сотрудников, диалогах создания (`AddEmployeeDialog`) и форме настроек.
  - В middleware (`proxy.ts`) для надежной защиты маршрутов (Админ -> `/dashboard`, IT-специалист -> `/it-portal`, Сотрудник -> `/portal`).
  - Для RLS-запросов создана безопасная PostgreSQL-функция `is_it_specialist` с флагом `SECURITY DEFINER`.

## 3. Выполненные архитектурные решения

В ходе доработки проекта были внедрены лучшие практики разработки современных веб-приложений:

1. **Разделение прав доступа на уровне роутинга (Next.js Proxy)**.
2. **Транзакционная целостность и бизнес-валидация на уровне СУБД (PostgreSQL Triggers & Constraints)**.
3. **Безопасное хранение и удобный просмотр конфиденциальных ключей в UI**.
4. **Шаблонизация создания активов для минимизации ручного ввода**.

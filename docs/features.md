# Функции CoreAsset

**Тема практики №149** — Описание функционала системы по модулям

---

## Общая схема

```
Сотрудник → Рабочее место → Компьютер → Установленное ПО → Лицензия
                                       → История неисправностей
```

---

## Модуль A — Сотрудники

**Что делает:** управляет реестром сотрудников предприятия.

| Функция | Описание |
|---------|---------|
| Добавить сотрудника | Форма с полями: ФИО, должность, отдел, email, табельный номер |
| Просмотреть список | Таблица всех сотрудников с поиском по ФИО |
| Редактировать | Изменить любое поле карточки |
| Уволить | Статус → «Уволен»; рабочее место автоматически освобождается |
| Привязать к рабочему месту | Назначить сотрудника на конкретный ПК / кабинет |

---

## Модуль B — Оборудование

**Что делает:** ведёт инвентарный реестр компьютеров.

| Функция | Описание |
|---------|---------|
| Добавить ПК | Инвентарный номер, серийный номер, тип устройства, кабинет |
| Характеристики | CPU, GPU, RAM, накопители (хранятся как гибкое JSON-поле) |
| Статус жизненного цикла | Активен / В ремонте / Вакантен / Списан |
| Просмотреть список | Таблица с фильтрами по статусу и кабинету |
| Карточка ПК | Детальный вид: характеристики + сотрудник + ПО + история тикетов |
| Удалить | С подтверждением; каскадно удаляет привязанные тикеты |

---

## Модуль C — Лицензии ПО

**Что делает:** контролирует баланс лицензий и предупреждает об истечении.

| Функция | Описание |
|---------|---------|
| Справочник ПО | Список разрешённого к установке программного обеспечения |
| Создать пул лицензий | Ключ, тип (подписка / бессрочная), лимит активаций, дата окончания |
| Привязать ПО к ПК | Уменьшает счётчик пула на 1; при лимите — операция заблокирована |
| Открепить ПО | Возвращает активацию в пул |
| Алерт истечения | Красная метка на пулах с подпиской < 30 дней |
| Статистика | Сколько мест занято / свободно по каждому пулу |

---

## Модуль D — Инциденты (Service Desk)

**Что делает:** ведёт журнал технических неисправностей и плановых работ.

| Функция | Описание |
|---------|---------|
| Создать тикет | Привязка к ПК, тип (поломка / сбой ПО / ТО), описание, приоритет |
| Приоритеты | Низкий · Средний · Высокий · Критический |
| Жизненный цикл | Открыт → В работе → Исправлен |
| Архив | Закрытые тикеты нельзя удалить — остаются в истории ПК навсегда |
| История на карточке ПК | Все тикеты по конкретному компьютеру в хронологическом порядке |

---

## Модуль E — Сквозные функции

**Что делает:** поиск, фильтрация и аналитика поверх всех модулей.

| Функция | Описание |
|---------|---------|
| Глобальный поиск | По инвентарному номеру, ФИО, IP-адресу, MAC-адресу |
| Фильтрация | По статусу жизненного цикла ПК и по кабинету |
| Сортировка таблиц | По любому столбцу |
| Дашборд | Карточки: всего мест / активных / в ремонте / вакантных / критических тикетов / истекающих лицензий |
| Подтверждение удаления | Модальное окно на любое деструктивное действие |

---

## Роли пользователей

| Роль | Права |
|------|-------|
| **Администратор** | Полный доступ: CRUD + управление лицензиями + управление пользователями |
| **Техник** | Управление ПК, тикетами, сотрудниками; без удаления лицензий |
| **Руководитель** | Только просмотр и дашборд (read-only) |

# Реализованный функционал проекта CoreAsset (Отчёт для защиты)

Ниже представлен структурированный список всех функций, модулей, доработок интерфейса и системных решений, которые были успешно реализованы в рамках проекта системы учёта ИТ-активов **CoreAsset**.

---

## 💻 1. Общая архитектура и системный каркас
- **Современный стек:** Приложение построено на **Next.js 15 (App Router)**, **TypeScript**, **Tailwind CSS**, **shadcn/ui** и **Supabase**.
- **Макет интерфейса:** Реализован единый адаптивный макет с боковым меню навигации (Sidebar) и шапкой (Header).
- **Разделение прав доступа (Middleware):** Реализована защита роутов в `proxy.ts`. Неавторизованные пользователи перенаправляются на экран логина, а авторизованные автоматически распределяются по ролям.
- **Управление сессией:** Интегрирован полноценный выход из системы (Logout) через Server Actions.

---

## 🛡️ 2. Безопасность и СУБД (PostgreSQL / Supabase)
- **Строгая ролевая модель (RLS):** Временные небезопасные политики `Dev Access` полностью удалены. Настроены жесткие RLS-политики доступа:
  - *Администраторы / IT-специалисты:* Полный CRUD ко всем ресурсам.
  - *Сотрудники:* Могут видеть только свои компьютеры и создавать/просматривать только свои инциденты (`employee_id = auth.uid()`).
- **Атомарный учет лицензий (Триггеры):** Подсчет занятых мест (`used_seats`) перенесен из кода в базу данных с помощью триггеров на таблице `computer_licenses`. Это полностью исключило состояние гонки (Race Condition).
- **Защита от перегрузки лицензий:** На уровне СУБД добавлено CHECK-ограничение `CHECK (used_seats <= total_seats)`. Попытка выдать лицензию сверх лимита блокируется на уровне транзакции БД.
- **Безопасные запросы:** Создана хранимая PostgreSQL-функция `is_it_specialist` с флагом `SECURITY DEFINER` для корректной проверки ролей в RLS-политиках.

---

## 👥 3. Модуль А: Реестр сотрудников (Employees)
- **CRUD сотрудников:** Полное управление базой персонала.
- **Механизм увольнения (Dismiss):** При увольнении сотрудника автоматически освобождаются связанные с ним рабочие места и компьютеры.
- **Интегрированный поиск:** Текстовый поиск сотрудников по ФИО.
- **Ролевая система:** Возможность назначения сотруднику одной из трех ролей: `admin`, `employee`, `it_specialist`.

---

## 🖥️ 4. Модуль B: Реестр оборудования (Computers)
- **CRUD компьютеров:** Полное управление списком вычислительной техники.
- **Шаблоны конфигураций (`computer_templates`):** Реализован полноценный CRUD шаблонов сборок ПК. При добавлении нового компьютера администратор может выбрать шаблон, и характеристики железа (CPU, RAM, GPU, диски) предзаполнятся автоматически.
- **Управление софтом:** Прямо на карточке компьютера выведен список установленного лицензионного ПО с кнопками мгновенной установки новой лицензии и удаления текущей.
- **Фильтрация и сортировка:** Добавлены dropdown-фильтры по статусу жизненного цикла ПК и кабинетам расположения, а также сортировка столбцов таблицы.

---

## 🔑 5. Модуль C: Справочник ПО и Лицензий (Licenses)
- **CRUD лицензий:** Учет программного обеспечения и свободных/занятых ключей.
- **Безопасный просмотр ключей:** Лицензионные ключи выводятся с маской по умолчанию (`••••-••••-••••-••••`). Добавлена интерактивная кнопка-глаз для просмотра ключа и кнопка мгновенного копирования.
- **Контроль лимитов:** При достижении лимита свободных копий приложение визуально подсвечивает индикатор занятых мест красным цветом.

---

## 🚨 6. Модуль D: Инциденты и Портал IT-специалиста (Incidents)
- **Автоматизация подачи заявок:** Сотрудник при создании тикета видит заблокированное поле кабинета, которое автоматически подтягивается из его профиля.
- **Портал ИТ-специалиста:** 
  - Главная страница переименована в **«Заявки»** (вкладка навигации и заголовок баннера).
  - Реализованы быстрые действия: перевод тикета в статус «В работе» («Взять в работу») и «Решено».
  - **Детальный просмотр на весь экран:** Клик по карточке тикета открывает стильный модальный диалог со всей информацией по инциденту, отправителю и устройству, включая прокручиваемое поле полного описания.
  - **Разделение кликов:** Использование `e.stopPropagation()` на кнопках статуса гарантирует, что нажатие на них меняет статус тикета, но не открывает модальное окно деталей.

---

## 📊 7. Модуль E: Сводные данные (Dashboard)
- **Аналитический дашборд:** Вывод ключевых счетчиков оборудования (всего устройств, активно, в ремонте, на складе).
- **Сквозной глобальный поиск:** Поиск по всей базе активов в шапке сайта по инвентарным номерам, ФИО сотрудников, IP- и MAC-адресам.
- **Баннер инцидентов:** Сводный список критических тикетов и предупреждений об истекающих лицензиях.

---

## 🔐 8. Страница авторизации (Auth UI)
- **Демо-вход:** Секция быстрого входа переработана в симметричную вертикальную колонку.
- **Премиальный UI кнопок:** Единый размер кнопок, рамки, скругления, плавный hover-эффект и векторные иконки Lucide.
- **Цветовое разграничение:** 
  - Кнопка Администратора — красная;
  - Кнопка Сотрудника — зеленая;
  - Кнопка IT-специалиста — синяя.
- **Локальный индикатор загрузки:** При нажатии на конкретную кнопку спиннер загрузки и текст «Вход…» отображаются только на ней, в то время как другие кнопки временно блокируются.

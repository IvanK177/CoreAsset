## 📋 План оставшихся задач (приоритезация)

### 🔴 Критическое — блокирует КТ-4 и КТ-5

1. **`middleware.ts`** — Auth guard: перенаправление неавторизованных с `/dashboard/*` на `/login`

### 🟡 Важное — требуется для КТ-4 (Модуль E: сквозные функции)

2. **UI привязки ПО к ПК** — На карточке компьютера: кнопка «Добавить ПО» → выбор из справочника → проверка лимита лицензий
3. **UI открепления ПО** — На карточке компьютера: кнопка «Удалить» рядом с каждым установленным ПО → возврат активации в пул
4. **Фильтр по статусу/кабинету** — ComputerTable: dropdown-фильтры над таблицей
5. **Фильтр по статусу/приоритету** — IncidentsTable: dropdown-фильтры
6. **Поиск по ФИО** — EmployeeTable: текстовый поиск
7. **Глобальный поиск** — Dashboard/Header: поиск по инв. номеру, ФИО, IP, MAC
8. **Сортировка таблиц** — Клиентская сортировка по клику на заголовок столбца

### 🟢 Желательное — для полноты и защиты

9. **Информация о сотруднике на карточке ПК** — Показать привязанного сотрудника + ссылку на рабочее место
10. **Привязка сотрудника к рабочему месту из карточки** — Dropdown выбора рабочего места на странице сотрудника
11. **Функциональное тестирование** — Протокол тестирования всех модулей
12. **Деплой на Vercel** — Финальный деплой
13. **Презентация** — 10–14 слайдов для защиты

### ✅ Уже выполнено (ранее ошибочно отмечено как невыполненное)

- **Миграции Supabase** — SQL-миграция с RLS-политиками, constraints, indexes, triggers (`20260520_proper_rls_policies_and_missing_constraints.sql`)
- **Seed-данные** — 12 компьютеров, 10 сотрудников, 8 рабочих мест, 6 ПО, 5 пулов лицензий, 13 установок, 6 инцидентов (`supabase/seed.sql`)
- **RLS-политики** — Role-based RLS для всех 7 таблиц (admin/tech/readonly)
- **Logout** — Кнопка выхода в Header с server action `signOut`
- **Визуальное предупреждение лимита лицензий** — `text-destructive font-semibold` при `used_seats >= total_seats` + CHECK-constraint на уровне БД

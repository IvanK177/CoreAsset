## 📋 План оставшихся задач (приоритезация)

### 🔴 Критическое — блокирует КТ-4 и КТ-5

1. **`middleware.ts`** — Auth guard: перенаправление неавторизованных с `/dashboard/*` на `/login`
2. **Миграции Supabase** — Создать SQL-миграции для всех таблиц + индексы + RLS-политики
3. **Seed-данные** — Скрипт для заполнения тестовыми данными (10+ компьютеров, 5+ сотрудников, лицензии, тикеты)
4. **UI привязки ПО к ПК** — На карточке компьютера: кнопка «Добавить ПО» → выбор из справочника → проверка лимита лицензий
5. **UI открепления ПО** — На карточке компьютера: кнопка «Удалить» рядом с каждым установленным ПО → возврат активации в пул
6. **RLS-политики** — Row Level Security для всех таблиц (admin: full, technician: CRUD minus licenses delete, manager: read-only)

### 🟡 Важное — требуется для КТ-4 (Модуль E: сквозные функции)

7. **Фильтр по статусу/кабинету** — ComputerTable: dropdown-фильтры над таблицей
8. **Фильтр по статусу/приоритету** — IncidentsTable: dropdown-фильтры
9. **Поиск по ФИО** — EmployeeTable: текстовый поиск
10. **Глобальный поиск** — Dashboard/Header: поиск по инв. номеру, ФИО, IP, MAC
11. **Сортировка таблиц** — Клиентская сортировка по клику на заголовок столбца

### 🟢 Желательное — для полноты и защиты

12. **Информация о сотруднике на карточке ПК** — Показать привязанного сотрудника + ссылку на рабочее место
13. **Привязка сотрудника к рабочему месту из карточки** — Dropdown выбора рабочего места на странице сотрудника
14. **Logout** — Кнопка выхода в Header
15. **Визуальное предупреждение лимита лицензий** — Красный индикатор когда `used_seats >= total_seats` в списке пулов
16. **Функциональное тестирование** — Протокол тестирования всех модулей
17. **Деплой на Vercel** — Финальный деплой
18. **Презентация** — 10–14 слайдов для защиты

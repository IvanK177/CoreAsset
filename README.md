# CoreAsset

**Стратегическая архитектура и план внедрения современной системы управления ИТ-активами (ITAM): Технологический стек 2026 года**

---

## Макроэкономический и стратегический императив управления ИТ-инфраструктурой

В современной корпоративной среде управление физическими и цифровыми активами вышло далеко за рамки традиционных практик складского или бухгалтерского учета, превратившись в критически важный домен стратегического управления бизнесом. Макроэкономические данные и отраслевая аналитика показывают, что, несмотря на то, что цифровая трансформация остается главным приоритетом для ИТ-руководителей, только **48%** цифровых инициатив в настоящее время достигают или превосходят ожидаемые бизнес-результаты. Этот высокий уровень неудач часто напрямую коррелирует с неспособностью организации точно отслеживать, контролировать и оптимизировать свою базовую технологическую инфраструктуру. Управление ИТ-активами (Information Technology Asset Management, ITAM) выступает фундаментальным слоем, обеспечивающим видимость и управляемость аппаратного и программного обеспечения, необходимого для реализации высокоуровневых бизнес-стратегий.

Финансовые последствия неадекватного управления ИТ-активами носят масштабный характер. Потеря всего одного портативного вычислительного устройства, если учитывать стоимость установленных на нем лицензий на программное обеспечение, инструментов безопасности и потенциального риска компрометации корпоративных данных, может обойтись компании в сумму, превышающую **$1 500**. Экстраполяция этой цифры на крупную организацию с тысячами сотрудников демонстрирует масштабную утечку капитала.

## Характеристика и основные функции приложения CoreAsset

Для решения описанных выше бизнес-задач в рамках проекта разрабатывается программный продукт **CoreAsset** — современная учебно-практическая система, предназначенная для комплексного учета рабочих мест, ИТ-оборудования и программных лицензий.

CoreAsset ориентирован на специалистов ИТ-отделов, системных администраторов и цифровых кураторов, чьей задачей является автоматизация учета и информирование пользователей о доступных технологических ресурсах предприятия. Основная проблема, которую решает приложение — устранение хаоса в инвентаризации, предотвращение использования нелицензионного программного обеспечения и ведение прозрачной истории технического обслуживания оборудования.

В соответствии со спецификацией программного обеспечения, система CoreAsset включает следующий обязательный функционал:

- **Ввод и хранение данных:** Безопасное добавление новых учетных единиц (компьютеры, сотрудники, лицензии) с сохранением в реляционную базу данных PostgreSQL. Валидация форматов и обязательных полей.
- **Просмотр записей:** Отображение глобальных списков (инвентарные ведомости) и детальных карточек по каждому отдельному рабочему месту с привязкой установленного ПО.
- **Редактирование и обновление:** Изменение статусов жизненного цикла техники и параметров устройств.
- **Безопасное удаление:** Удаление записей с механизмом подтверждения и защитой от нарушения целостности базы данных (недопущение появления «осиротевших» записей).
- **Поиск, фильтрация и сортировка:** Инструменты для быстрого нахождения техники по серийному номеру или ФИО сотрудника, а также фильтрация оборудования по статусу (например, «в ремонте»).
- **Генерация отчетности:** Формирование итоговой аналитики, статистических сводок по количеству доступных и занятых программных лицензий.

## Сравнительный анализ аналогов и рыночной экосистемы ITAM

Рыночная экосистема ITAM в 2026 году строго сегментирована между закрытыми проприетарными корпоративными платформами (ServiceNow CMDB, Microsoft Endpoint Manager, Dynatrace) и разнообразными решениями с открытым исходным кодом.

В ходе анализа предметной области были выделены следующие основные аналоги разрабатываемой системы CoreAsset:

| Аналог | Плюсы | Минусы |
|--------|-------|--------|
| **Snipe-IT** | Один из самых популярных open-source аналогов. Интуитивно понятный интерфейс и отличные функции отслеживания жизненного цикла активов. | Написан на PHP, что может замедлять внедрение современных реактивных компонентов; иногда перегружен избыточными функциями для небольших отделов. |
| **GLPI** | Мощная комбинированная система ITAM и ITSM (Helpdesk). Позволяет связывать активы напрямую с заявками в службу поддержки. | Сложный, громоздкий интерфейс, требующий длительного обучения персонала. |
| **Ralph** | Отлично подходит для центров обработки данных. Визуализация серверных стоек. | Слишком сложна для простого учета офисных рабочих мест. |

**Уникальные особенности CoreAsset:** В отличие от громоздких аналогов, CoreAsset предлагает сфокусированный, молниеносно работающий интерфейс (за счет Next.js и React Server Components), строгую API-first архитектуру (на базе GraphQL) и надежную транзакционную защиту от превышения лимита программных лицензий при их назначении на рабочие места.

## Теоретические основы ИТ-архитектуры: Демаркация Enterprise и Software Architecture

Для проектирования надежного решения ITAM необходимо установить четкое теоретическое и практическое разграничение между корпоративной архитектурой (Enterprise Architecture) и архитектурой программного обеспечения (Software Architecture).

- **Корпоративная архитектура** функционирует на макроуровне, согласовывая общую бизнес-стратегию со всем ИТ-комплексом организации в целом.
- **Архитектура программного обеспечения** фокусируется на микроуровневом проектировании отдельных систем и взаимодействии их составных компонентов во время выполнения (runtime).

Разработка высокопроизводительного сервиса CoreAsset требует строгой, масштабируемой программной архитектуры для безупречного выполнения более широких мандатов корпоративной архитектуры.

## Доменно-ориентированное проектирование (DDD) логики «Рабочего места»

Система должна осуществлять комплексный учет компьютеров, их характеристик, установленного ПО, лицензий и истории неисправностей. Логическая концепция **«Рабочего места» (Workplace)** выступает в роли центрального агрегата в предлагаемой архитектуре. Это агрегирующая, абстрактная транзакционная концепция, которая семантически связывает воедино разрозненные физические и цифровые элементы:

```
Пользователь → Рабочее место → Аппаратный актив → Установленное ПО → Юридическая лицензия
```

## Детализированная архитектура базы данных и транзакционное управление

Аритектура хранения данных спроектирована в строгом соответствии с требованиями третьей нормальной формы (3NF). Взаимодействие с реляционной базой данных осуществляется через **Prisma ORM**, обеспечивающую строгую типизацию запросов на основе файла `schema.prisma`.

| Таблица (Сущность) | Описание и бизнес-логика | Ключевые атрибуты и типы |
|---------------------|--------------------------|--------------------------|
| `Hardware_Assets` | Центральный реестр физического оборудования | `id` (UUID, PK), `serial_number`, `lifecycle_state`, `discovery_metadata` (JSONB) |
| `Users` | Профили сотрудников | `id` (UUID, PK), `employee_id`, `email` |
| `Workplaces` | Связующее звено актива и сотрудника в локации | `id` (UUID, PK), `user_id`, `hardware_id` |
| `Software_Catalog` | Справочник утвержденного ПО | `id` (UUID, PK), `software_name`, `version` |
| `Licenses` | Управление финансовыми аспектами ПО | `id` (UUID, PK), `concurrency_limit` (INT) |
| `Software_Installations` | Точка контроля комплаенса (M:N) | `id` (UUID, PK), `hardware_id`, `license_id` |
| `Fault_History_Incidents` | Журнал технического обслуживания | `id` (UUID, PK), `hardware_id`, `incident_description` |

## Современный технологический стек (Бесплатная облачная инфраструктура)

Для реализации данного плана без финансовых затрат (**0 рублей**) выбран современный стек, опирающийся на облачные сервисы с generous free-tier (щедрыми бесплатными лимитами), обеспечивающими работу полноценного Enterprise-подобного приложения.

### 1. База данных: Supabase (PostgreSQL)

Используется Supabase в качестве полностью управляемой бессерверной СУБД PostgreSQL. Бесплатный тариф предоставляет:

- **500 МБ** дискового пространства
- **1 ГБ** хранилища файлов
- **5 ГБ** исходящего сетевого трафика
- Поддержка до **50 000** активных пользователей в месяц

Этого лимита с огромным запасом хватит для хранения сотен тысяч текстовых записей об ИТ-активах предприятия в рамках практики. Взаимодействие осуществляется через Prisma ORM.

### 2. Frontend-приложение: Next.js 15 + Vercel

Пользовательский интерфейс построен на **Next.js 15** (с App Router) и компонентах **Shadcn UI / Tailwind CSS**. Управление клиентским состоянием делегировано **TanStack Query v5**. Развертывание фронтенда осуществляется на платформе **Vercel**.

Бесплатный тариф Hobby включает:

- **100 ГБ** пропускной способности в месяц
- Автоматизированный CI/CD пайплайн (развертывание прямо из GitHub при пуше в ветку `main`)
- Глобальную CDN

### 3. Backend API: NestJS + Koyeb (или Vercel)

Слой бизнес-логики и GraphQL API написан на **NestJS**. Для бесплатного развертывания бэкенда используется платформа **Koyeb**, которая предоставляет:

- Контейнер с **1 ГБ** оперативной памяти и **0.25 vCPU** на своем бесплатном тарифе

Это идеально подходит для непрерывной работы Node.js сервера. В качестве альтернативы можно скомпилировать NestJS как бессерверную функцию и также бесплатно развернуть на Vercel.

### 4. Кэширование (Опционально): Upstash Redis

Для кэширования тяжелых агрегационных запросов и управления сессиями интегрируется **Upstash** — Serverless Redis, который предлагает отличный бесплатный тариф, не требующий привязки банковской карты и интегрирующийся с Koyeb и Vercel.

## Мини-документация (Quick Start Guide)

- **Название проекта:** CoreAsset (ITAM Workspace Suite)
- **Описание:** Комплексная система учета рабочих мест, ИТ-оборудования и программных лицензий.

### Требования для локальной разработки

- **Node.js** (v20 LTS или выше)
- **Пакетный менеджер pnpm** (рекомендуется для монорепозиториев Turborepo)
- **Docker Desktop** (для запуска локальной БД, если не используется облачная)

### Структура монорепозитория (Turborepo)

```
coreasset-itam/
├── apps/
│   ├── web/          — Фронтенд на Next.js 15 (порт 3000)
│   └── api/          — GraphQL бэкенд на NestJS (порт 4000)
├── packages/
│   ├── database/     — Схемы Prisma, миграции и сиды (seeds)
│   └── ui/           — Общие UI-компоненты Shadcn
```

### Инструкция по локальному запуску

**1. Клонирование репозитория:**

```bash
git clone https://github.com/your-username/coreasset-itam.git
cd coreasset-itam
```

**2. Установка зависимостей во всех пакетах:**

```bash
pnpm install
```

**3. Настройка переменных окружения:**

Скопируйте пример конфига в корне проекта и в папке базы данных:

```bash
cp .env.example .env
# Вставьте ваш Connection String от базы данных Supabase в DATABASE_URL
```

**4. Инициализация базы данных:**

Синхронизируйте структуру Prisma с вашей подключенной облачной БД Supabase:

```bash
pnpm --filter database run db:push
# (Опционально) Заполнить БД тестовыми данными оборудования
pnpm --filter database run db:seed
```

**5. Запуск проекта:**

Turborepo параллельно запустит и Next.js, и NestJS:

```bash
pnpm run dev
```

- UI доступен по адресу: **http://localhost:3000**
- GraphQL Playground доступен по адресу: **http://localhost:4000/graphql**

### Процесс облачного развертывания (Production)

**База данных (Supabase):**
Создайте новый проект на [Supabase.org](https://supabase.org), скопируйте Transaction connection string из настроек Database и сохраните его.

**Backend (Koyeb / Vercel):**
Подключите ваш GitHub аккаунт к Koyeb, выберите репозиторий, укажите:
- Build Command: `pnpm run build --filter api`
- Run Command: `pnpm run start:prod --filter api`
- Добавьте `DATABASE_URL` в Environment Variables.

**Frontend (Vercel):**
Импортируйте проект в Vercel, выберите папку `apps/web` как Root Directory. Добавьте переменную `NEXT_PUBLIC_GRAPHQL_API_URL`, указывающую на домен задеплоенного бэкенда в Koyeb. Нажмите **Deploy**.
